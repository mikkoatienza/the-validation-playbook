You ran a small test and got a signal. That is good. But one signal is not
fit. Fit shows up when people return without you begging them.

This module helps you measure early product‑market fit (PMF) signals before
you scale. You are not hunting vanity numbers. You are hunting repeat use.

**Output for this module**
- A PMF Signal Checklist score
- A simple momentum loop sketch
- Notes on reactivation and repeat value

## PMF Signals (Plain Language)
Product‑market fit means the product solves a real problem so well that people
keep using it without persuasion. Not forever. Not perfectly. But enough that
the habit starts to form.

Early PMF signals are simple:
- People come back.
- People finish the core action.
- People tell others without being asked.
- People would be disappointed if it went away.

If those signals are weak, you may not have fit yet. You may only have
curiosity.

**Activation vs retention**
Activation is the first use. Retention is the second use. Many products look
good at activation and still fail because people do not return. You can fix
activation with copy and onboarding. You fix retention with real value.

**What “come back” means**
Repeat use depends on the natural cycle of the problem. The cycle tells you
how soon a return should show up:
- Daily problems: you should see returns within a day or two.
- Weekly problems: you should see returns within a week.
- Monthly problems: you should see returns within a month.

If your product solves a weekly problem, a daily return is not required. But a
monthly return is too slow. Match your expectations to the pain cycle.

**What counts as repeat use**
- They run the core action again without a prompt.
- They save it and return on their own.
- They pull a teammate in because it helps them repeat the workflow.

If you only see repeat use after reminders, the loop is still weak.

**Time‑to‑value (plain language)**
If the core win takes ten steps, people leak before they feel the payoff.
Shorter time‑to‑value usually means more returns.

**Completion (plain language)**
Completion is whether they finish the action that creates the result.
If they stop halfway, the value often never lands.

**Reactivation (plain language)**
Reactivation is what happens after someone drops. A good product can bring
some of them back with one clear reminder or nudge.

**Where to get early evidence**
- Use the last 10 users who touched the MVP test.
- If you only have 5, use those 5 and note the small sample.
- Write down what actually happened, not what you hoped.

Small samples are normal. Just be honest about them.

## The PMF Signal Checklist
Use this checklist to score your early fit signals. This is not a metric
dashboard. It is a field test.

**PMF Signal Checklist (score 1–5)**
1. **Repeat use**: Do people return on their own within the expected cycle?
2. **Time‑to‑value**: Do they reach the “aha” moment fast?
3. **Completion**: Do they finish the core action?
4. **Reactivation**: If they drop off, can they be pulled back?
5. **Referrals**: Do they recommend or invite others without a prompt?
6. **Willingness to pay again**: Would they renew or extend the trial?

Score each 1–5 based on evidence, not hope. A “3” is not a failure. It is a
signal to adjust.

**How to score 1–5 (quick anchors)**
- 1 = almost never
- 3 = sometimes, inconsistent
- 5 = often, predictable

Think of these as rough anchors, not math. Your goal is direction, not
precision.

**Evidence examples by signal**
- Repeat use: 3 out of 10 return in the expected cycle.
- Time‑to‑value: most users reach the core result in one session.
- Completion: people finish the task you designed for them.
- Reactivation: a short follow‑up brings some back to the core action.
- Referrals: users mention it or invite a teammate without incentives.
- Pay again: buyers ask to continue or extend the pilot.

If you cannot name evidence, score it a 1 for now.

**Score interpretation (early)**
- Mostly 1–2: you do not have fit yet.
- Mostly 3: you have partial fit; the loop needs work.
- Several 4–5: you are close; tighten the weak signal.

Do not chase a perfect score. Chase a consistent signal.

**Optional sanity check**
Ask: “How disappointed would you be if this disappeared?”
- Very disappointed
- Somewhat disappointed
- Not disappointed

If “very disappointed” is rare, retention needs work.

## A Real Example (Duolingo Streaks)
Many language apps see a spike in sign‑ups, then a drop in daily practice.
People like the idea but stop returning. Duolingo is known for building habits
through streaks, reminders, and small daily goals. Those features do not make
the first lesson better. They make the second lesson happen.

The insight is simple: repeat use is not automatic. It is designed.

Your product may not need streaks. But it does need a reason to return that
fits the problem cycle. If the pain is weekly, your loop should be weekly.

If the product only works when you push it, it is not yet a product. It is a
project you are managing by hand.

The lesson is not “add streaks.” The lesson is “build a reason to return.”
If the reason is missing, you will keep spending energy on reminders that do
not stick.

Diagram cue: a simple retention curve or loop sketch (first use -> return -> habit).

## The Momentum Loop (Sketch It)
A momentum loop is a simple cycle that turns one use into the next use. It is
not a complex funnel. It is a small loop you can explain in one sentence.

**Loop template**
Trigger → Action → Result → Reason to return

Example (generic):
- Trigger: weekly compliance checklist
- Action: run the report
- Result: missed items found
- Reason to return: risk avoided and report saved

If you cannot describe this loop, you do not yet know why people should return.

**Loop strength questions**
- Does the result create a new reason to act again?
- Can the user feel the win without you explaining it?
- Is the reason to return tied to the problem, not your product?

If the loop depends on reminders only, it is fragile.

**Common return triggers**
- A calendar moment (end of week, end of month)
- A risk event (deadline, audit, renewal)
- A threshold (inventory low, lead quality drop)

If you cannot name a trigger, you do not have a loop. You have a hope.

**Retention curve (plain language)**
Imagine a simple curve: day one is high, then it drops. If the line keeps
dropping to zero, there is no habit. If it flattens, even at a lower level,
you have a base of repeat use. That flat line is early fit.

Example (generic):
If 10 users try the product and 3 return the next week, that is not a win
yet, but it is a base you can grow. If 0 return, the loop is broken.

## Common Traps
- Celebrating sign‑ups instead of repeat use
- Confusing a one‑time spike with a habit
- Ignoring churn because a few users are loud
- Measuring engagement but not completion
- Forcing referrals before the loop works
- Treating reactivation as retention
- Shipping new features to avoid fixing the loop

Chasing growth before repeat use is like pouring water into a bucket with a
hole. Fix the leak first.

If you are tempted to celebrate a spike, compare it to return rates. A spike
without returns is noise, not momentum.
