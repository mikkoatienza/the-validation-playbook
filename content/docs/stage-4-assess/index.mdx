<Callout type="idea" title="The promise of this stage">
By the end of Stage 4, you will have **an MVP test result with a clear pass/fail**, **a PMF signal score with a loop sketch**, and **a focused iteration path** (improve, pivot, or stop). If you cannot get repeat use, you likely do not have fit yet.
</Callout>

Stage 3 tested whether people will commit. Stage 4 tests whether they come back. This is where early fit appears. A loop is the trigger-action-return pattern that brings people back. If the loop is weak, everything breaks here.

## The Three Modules

Each module narrows the signal. If Module 1 is weak, Module 2 is noise. If Module 2 is unclear, Module 3 is guesswork. Do them in order.

## Stage Rules

<Callout type="warn" title="Keep it honest">
- A test is not a product. Smallest proof beats biggest build.
- Repeat use beats signups. Behavior beats buzz.
- Change one variable at a time. Otherwise you learn nothing.
- If the loop is weak, fix the blocker, not the distracting feature.
</Callout>

The goal is not growth. The goal is repeatable proof.

<Callout type="error" title="Common traps">
- Building too much before a pass line
- Celebrating signups instead of repeat use
- Changing three things at once
- Ignoring churn because a few users are loud
- Fixing symptoms instead of the loop
</Callout>
