title: "Stage 4: Assess"
description: "Build minimal proof, measure signal. Run the smallest experiment that tests the riskiest assumption."
---

Stage 3 tested whether people will commit. Stage 4 tests whether they come back.  
This is where early fit appears.  
If the loop is weak, everything breaks here.

A loop is the trigger‑action‑return pattern that brings people back.

## Stage Promise
By the end of Stage 4, you should have:

- An MVP (minimum viable product) test result with a clear pass/fail
- A PMF (product‑market fit) signal score and a simple loop sketch
- A focused iteration path (improve, pivot, or stop)

If you cannot get repeat use, you likely do not have fit yet.
Keep the loop small and real.

## What This Stage Produces
Stage 4 takes your committed buyers and tests whether the product creates
repeatable value. You will build three artifacts that turn commitment into fit:

- **Experiment Card + MVP results**
- **PMF Signal Checklist + loop sketch**
- **Iteration Decision Note** (next change + pass line)

These are the proof points that justify the commitment decision in Stage 5.

## Stage Rules (Keep It Honest)
This stage works best when you follow a few rules:

- A test is not a product. Smallest proof beats biggest build.
- Repeat use beats signups. Behavior beats buzz.
- Change one variable at a time. Otherwise you learn nothing.
- If the loop is weak, fix the blocker, not the distracting feature.

The goal is not growth. The goal is repeatable proof.

## How the Modules Fit
This stage has three modules, each with a specific activity:

1. **Run the smallest test that proves demand** (Activity: MVP Sprint)  
   You test demand with the smallest possible build.

2. **Check if people come back and use it again** (Activity: Fit Signals)  
   You measure repeat use, time‑to‑value, and early retention.

3. **Fix what blocks fit, not what is shiny** (Activity: Fit Fixes)  
   You choose one change and test it, not five.

Each module narrows the signal. If Module 1 is weak, Module 2 is noise. If
Module 2 is unclear, Module 3 is guesswork. Do them in order.

## What to Ignore for Now
Stage 4 is not the time for:

- Full product buildouts
- Scaling acquisition
- Hiring growth teams
- Vanity dashboards or decorative metrics

Those moves depend on fit signals you do not have yet.

## Common Traps (Avoid Them Early)
Most founders get stuck in one of these:

- Building too much before a pass line
- Celebrating signups instead of repeat use
- Changing three things at once
- Ignoring churn because a few users are loud
- Fixing symptoms instead of the loop

If you recognize one, stop and reset. The goal is not speed. The goal is
signal.

## Pace and Volume
Speed matters here, but only if the evidence stays clean. A good rhythm is:

- One MVP test in the next two weeks
- One PMF signal checklist scored from real usage
- One focused iteration test with a pass line

Avoid over‑polishing. Your job is to learn fast, not to ship perfect.

## Evidence Standard (What Counts)
You do not need scale, but you do need real signals:

- A pass/fail MVP result against a clear line
- Repeat use within the natural problem cycle
- A loop you can explain in one sentence

If you do not have those signals, pause and fix the gap.

## Evidence Depth (Strong vs. Weak)
Not all signals are equal. Use this lens:

- **Weak**: curiosity, compliments, “cool idea”
- **Medium**: repeat use with reminders, partial completion
- **Strong**: repeat use without prompts, clear loop trigger

Stage 4 is about moving toward strong signals. If you are stuck in weak
signals, avoid moving on.

## Reality Check (Fast)
Ask yourself:

- "Do people return without reminders?"
- "Can I explain the loop in one sentence?"
- "Did the last change improve the signal?"

If any answer is no, slow down and fix it. This stage is cheaper than scaling
a weak loop.

## What Success Feels Like
By the end of this stage, you should be able to say, “This test passed, this
loop exists, and this is the next fix.” If you cannot, you are not ready to
commit.

Stage 4 is your filter. It protects you from scaling a weak loop.

## Handoff to Module 1
Start with the smallest test. If the MVP does not pass, nothing else matters.
Module 1 begins with the MVP Sprint.

One last reminder: repeat use is the compass.
