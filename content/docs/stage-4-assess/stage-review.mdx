---

You entered this stage with commitment signals. You leave it with evidence of
repeat use and a clear path forward. That is the shift. Stage 4 is where you
stop guessing about fit and start acting on proof.

## The Three Artifacts That Matter
If you completed Stage 4 as designed, you now have:

- An MVP test result with a clear pass line
- A PMF signal score and loop sketch
- A focused iteration decision (improve, pivot, or stop)

That is the minimum foundation for Stage 5.

## Readiness Gate (30 Seconds)
Use this compact gate to decide if you should move on:

- **MVP Sprint:** pass/fail result tied to a clear pass line
- **Fit Signals:** repeat use within the problem cycle
- **Fit Fixes:** one focused change with a test plan
- **Loop Statement:** “They return because [trigger] → [action] → [result].”

If any box is empty, slow down and fill it.

## Readiness Questions (Fast)
Ask yourself:

- "Did repeat use show up without heavy prompting?"
- "Can I explain the loop in one clean sentence?"
- "Do I know the next test and why it matters?"

If any answer is no, tighten the evidence before you move forward.

## If the Evidence Is Thin
Avoid forcing the next stage. Use one of these fixes first:

- Shrink the test and rerun it
- Tighten the loop trigger and retest returns
- Drop the weakest segment and focus on the strongest
- Change one variable and rerun the test

The goal is not to feel good. The goal is to get real.

## Module Recap (What Each One Gave You)
Stage 4 worked because each module did a specific job:

- **MVP Sprint** proved demand with the smallest possible test.
- **Fit Signals** showed whether people returned and why.
- **Fit Fixes** chose one change to improve the loop.

If any module feels weak, fix it before you move on. A weak loop will collapse
under the pressure of Stage 5.

## Stage 4 Retrospective (Two Questions)
Write short answers before you move forward:

- Which signal improved the most and why?
- Which blocker refused to move and why?

That reflection will make your commitment decision cleaner in Stage 5.

## Decision Summary (One Sentence)
Write this line and keep it with your artifacts:

"They return because [trigger] leads to [action] and [result]."

If you cannot write that line, the loop is not stable.

## Evidence Snapshot (One Paragraph)
Before you move on, write a single paragraph that captures:

- The strongest repeat‑use signal you observed
- The clearest loop trigger you can name
- The change that moved the signal

Keep it tight. If you cannot write it in one paragraph, the evidence is still
too fuzzy. Tighten it now while the tests are fresh.

## Carry‑Forward Signals
Before you enter Stage 5, underline the strongest proof in your notes:

- The strongest repeat‑use signal
- The clearest loop trigger
- The change that moved the signal

Those signals should anchor your commitment decision. Keep them visible.

## If Two Signals Disagree
Sometimes repeat use looks strong but the loop is unclear. Use this tie‑breaker:

- Trust the signal closest to behavior, not opinion
- Prefer repeatable returns over one‑time spikes
- Re‑run the weakest test with a tighter pass line

If the signals still conflict, slow down and retest. Mixed evidence is rarely
a green light.

## If You Want to Move Faster
Speed is useful only when the evidence keeps up. Use these safe shortcuts:

- Retest the weakest loop trigger with a tighter pass line
- Narrow to the segment that returns without reminders
- Drop the lowest‑engagement workflow and focus on the strongest

Moving fast means narrowing the focus, not skipping the proof.

## After You Leave This Stage
You will feel pressure to scale. Resist the urge to celebrate too early.
Stage 5 is where you decide whether to commit, not where you accelerate.

If you skip this discipline, you may scale a loop that collapses under load.
Stage 4 is the cheaper place to be skeptical.

## The Handoff to Stage 5: Lock
Stage 5 will ask you to decide whether to commit or stop, and whether you are
the right founder for the fight. That work only makes sense if Stage 4 is real.

You are about to move from "Do they come back?" to "Do we commit?"

## Closing Line
No repeat use, no fit.
