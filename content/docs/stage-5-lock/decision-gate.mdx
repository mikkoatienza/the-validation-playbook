You have evidence now. The only thing left is the decision. Most founders
avoid this moment because it feels final. It is not. It is simply honest.

This module turns your evidence into a clear go / pivot / stop call.
No drifting. No limbo. Decide and move.

**Output for this module**
- A documented go / pivot / stop statement
- A decision grid with evidence notes
- A short commitment plan or stop rationale

## The Decision Gate (Plain Language)
The decision gate is a structured pause where you look at evidence and choose
one path. It is not a vibe check. It is not a pep talk. It is a call based on
what the data has earned.

If you avoid the gate, the market will decide for you.

**Decision vs delay**
Delay feels safe, but it steals momentum. A decision creates a path.
If you are stuck, you are already paying the cost.

**What counts as a decision**
- “Go” with a clear 30‑day focus
- “Pivot” with a new outcome and a fresh test
- “Stop” with a written reuse plan

If you cannot say which one, you have not decided yet.

**Decision gate checklist**
- Evidence reviewed
- Weakest signal named
- One more test run if needed
- Decision written in one sentence

**Decision window**
Give yourself a short window to decide (7–10 days). If it drags, the decision
turns into avoidance.

**Alignment check**
If you have a co‑founder, review the grid together. If you disagree, write
down why. The disagreement is also evidence.

**Decision meeting (15 minutes)**
- Review the evidence snapshot
- Name the weakest signal
- Choose the path
- Write the next action and date

Keep it short. Long meetings usually mean the evidence is weak.

## The Decision Grid
Use a simple grid to force the choice. You are not picking what you want.
You are picking what the evidence allows.

**Decision grid**
| Path | Evidence | Decision |
| --- | --- | --- |
| Go | | |
| Pivot | | |
| Stop | | |

**Decision grid rule**
You must write evidence in each row before you pick a path. If you can’t fill
a row, you don’t have enough signal for that path.

Write 1–2 evidence bullets per row. If a row only has wishes, leave it blank.

**Grid example (generic)**
- Go: repeat use is rising + paid pilots renew
- Pivot: users finish, but they ask for a different outcome
- Stop: repeat use near zero after two focused fixes

**Second pass**
Circle the path with the most evidence. If it isn’t the path you want, pause.
Your desire is not the signal. The grid is.

**Go means**
- Repeat use exists and is improving
- Buyers accept the price or re‑buy
- Channel is predictable enough to reach buyers

**Pivot means**
- Signals are weak, but a different outcome has pull
- The loop is broken, not the problem
- Evidence suggests a better segment or workflow

**Stop means**
- Repeat use is near zero after focused tests
- Pricing fails even after a clean offer
- Channel access is too costly to sustain

**What each path commits you to**
- Go: focus the build around the proven loop
- Pivot: a new outcome and a fresh test, not a rename
- Stop: closing the loop and freeing your time

**Signal weighting (simple)**
Repeat use usually matters most, then pricing, then channel. If repeat use is
weak, a “go” is fragile even if everything else looks good.

If the evidence is mixed, do not guess. Run one more focused test and decide
after it. One more week beats six more months.

**Decision signal**
If your strongest signal is still weak, stopping may be the best move. A weak
“go” is often a slow “stop.”

If that feels harsh, remember the goal: protect your time and energy.

If the signals are mixed, your default should be one more focused test.

If the signals are strong, move. Momentum fades fast when you hesitate.

Indecision is often slow failure.

**Guardrail**
Do not let a single loud user overpower silent evidence. Volume matters more
than enthusiasm.

**Go‑small rule**
If you feel pulled to “go” but the evidence is thin, reduce the commitment
and run a final test. A small go is still a test, not a launch.

## A Real Example (Mailchimp’s Decision to Commit)
Mailchimp started as a side project inside a web design agency. Over time, the
email tool grew while the agency work stalled. The founders decided to commit
to Mailchimp and stop the agency path. They chose the product that had pull.

The lesson is not “pick software.” The lesson is “choose the path with proof.”

They were not chasing a trend. They were following the signal that customers
kept using and paying for.

They also cut the distraction that slowed them down. Commitment is a focus
decision, not a finish line.

They said no to good work to say yes to the work with evidence.

## Exercise: Evidence Review + Go / Pivot / Stop Statement
Before you decide, take one clean pass through the evidence. The goal is not
to collect more data. The goal is to see what you already earned:

- MVP test results (pass/fail)
- PMF signal score (repeat use, time‑to‑value, completion)
- Channel access (cost per conversation)
- Pricing signal (paid pilots or renewals)

If any area is missing, mark it as weak. A missing signal is still a signal.

**Evidence risk check**
Ask: “What would I have to assume to say ‘go’?”
If the answer is “a lot,” you are not ready.

**Quick scoring (1–5)**
- Problem strength
- Repeat use
- Pricing signal
- Channel access

Circle the weakest. That is where doubt lives.

**Total score (optional)**
Add the four scores. If you are under 10, you likely have more to fix than
to scale. If you are around 12–14, run one more focused test. If you are
above 15, “go” is usually earned.

**Missing evidence rule**
If one box is empty, you probably do not have a “go” yet. Fill it or pause
the decision.

**Evidence audit (2 minutes)**
Write one line per box:
- Problem: what proof do I have?
- Repeat use: how many returned?
- Pricing: who paid or committed?
- Channel: what was the cost per conversation?

**Contradiction check**
If one signal is strong and another is weak, write the conflict. Decide which
signal you trust more and why.

**Red flags**
- Returns happen mostly after you push
- Price only works with heavy discounts
- Channel only works through favors

**Evidence snapshot (one paragraph)**
Write 3 to 5 sentences that summarize the evidence as if you were explaining
it to a skeptical partner. If you can’t explain it, you don’t own it.

Example (generic):
“Ten clinics tested the pilot. Three returned the next week without a prompt.
Two paid again. Channel cost was $40 per conversation. Our weakest signal is
repeat use, so the next test will target time‑to‑value.”

**Decision boundary**
Write one line: “If the weakest signal does not improve by next month, I will
___.” This keeps you from drifting back into limbo.

Example (generic):
“If repeat use is still below 3 of 10 next month, I will stop.”

If you cannot write this line, you have not really decided.

**Decision statement**
- We choose: Go / Pivot / Stop
- Because: (evidence summary in one sentence)
- Next step: (commitment action or stop action)

Example (generic):
- We choose: Go
- Because: repeat use is rising and two pilots renewed at the target price.
- Next step: commit 30 days to the core loop and track weekly return.

**Decision confidence (1–5)**
Write a number. If it is below 3, run one more test before you commit.

**Confidence calibration**
If your confidence is 4 or 5, be able to name the two strongest signals and
the one weakest signal you are actively fixing.

Keep it short. If you cannot explain the decision in one sentence, you do not
understand it yet.

**One‑sentence proof**
Write: “We choose ___ because ___.” If you can’t finish it, you’re not ready.

**Commitment plan (if you go)**
- The next 30‑day focus
- The single metric you will watch weekly
- The team or time you will commit

Keep the focus narrow. Your job is to protect the strongest signal and raise
the weakest one.

Example (generic):
“Focus: shorten time‑to‑value. Metric: week‑one return. Commitment: 10 hours
per week and 5 user check‑ins.”

If the plan doesn’t fit your current week, it isn’t a real “go.”

**Pivot plan (if you pivot)**
- What outcome you will now target
- Which signal you are trying to fix first
- The first test you will run

If your pivot plan cannot name a new outcome, it is not a pivot.

Pivot guardrail: keep the strongest signal and change only the broken part.

**Timebox**
Set a 30‑day window for the decision path. If you cannot show progress in a
month, revisit the gate.

**Accountability note**
Write who will review the results with you. Decisions stick when someone
expects the update.

**First‑week action**
Write the first action you will take within 72 hours. Decisions are only
real if they change your calendar.

## Evidence Artifact (Decision Note)
Capture the decision in one place:

- Decision chosen
- Evidence summary (3 bullets)
- Risks accepted
- Next action (or stop plan)

**Decision Note (simple)**
| Item | Notes |
| --- | --- |
| Decision | |
| Evidence | |
| Risks | |
| Next action | |

Write three evidence bullets under “Evidence.” If you can’t, you’re still
deciding by feel.

Example (generic):
- Decision: Go
- Evidence: repeat use rising, two renewals, channel cost stable
- Risks: weak referral signal
- Next action: 30‑day loop focus + weekly return tracking

**Stop plan (if you stop)**
- What you will reuse
- What you will stop spending time on
- The date you will close the loop

This is your proof that you decided based on evidence, not hope.

If you cannot write the stop plan, you may not be ready to stop.

**Risk note**
Write the risk you accept if you go. Every “go” is a bet.

**Decision date**
Write the date you made the call. It keeps you honest if you revisit later.

**Revisit trigger**
Write the condition that would reopen this decision (e.g., repeat use drops,
channel cost doubles). Decide it now, not in panic later.

Example (generic):
“Revisit if week‑one return drops below 2 of 10.”

## Common Traps
- Moving the goalposts because the answer is uncomfortable
- Calling “pivot” when you really mean “avoid stop”
- Choosing “go” because you are tired
- Ignoring a weak signal because one strong signal felt good
- Letting ego override the data
- Hiding behind “one more test” without a pass line
- Mixing go and pivot in the same week
- Letting sunk cost speak louder than evidence

If you cannot point to evidence, you are not ready to say “go.”

If you feel relief instead of clarity, you are still avoiding the decision.

If you keep rewriting the evidence to make it feel better, stop and reset.
